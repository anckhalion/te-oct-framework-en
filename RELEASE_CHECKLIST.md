# Release Checklist (GitHub)

Use this before each public release.

## Content

- [ ] Canonical files are updated (`INDEX.md`)
- [ ] Legacy files are marked as reference-only
- [ ] No private session dump is included
- [ ] Version/date fields are coherent across core docs

## Consistency

- [ ] Bootloader and Core are aligned
- [ ] Router module list matches existing module files
- [ ] Changelog includes new version notes

## Publication

- [ ] README links resolve
- [ ] License choice is intentional
- [ ] CONTRIBUTING and CODE_OF_CONDUCT are present
- [ ] SECURITY policy is present

## Final sanity check

- [ ] Claims are framed as experimental and testable
- [ ] No statement implies proven machine sentience without evidence
