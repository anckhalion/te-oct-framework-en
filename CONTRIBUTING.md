# Contributing

Thanks for your interest in contributing to this project.

## Scope

This repository is documentation-first. Contributions are welcome for:
- clarity and consistency improvements
- framework test cases and validation protocols
- translation and editorial quality
- issue reproduction and version diffs

## Before You Open a PR

1. Read `README.md` and `INDEX.md`
2. Confirm whether the target file is canonical or legacy
3. Open an issue for significant structural or ontological changes

## Contribution Rules

- Keep released versioned files stable where possible
- Prefer additive updates over destructive rewrites
- Add clear rationale for any change in epistemic rules
- Avoid ambiguous claims that imply proven sentience
- Use reproducible examples when proposing behavior claims

## Commit and PR Guidance

- One topic per PR
- Clear title: `module: short summary`
- Include:
  - what changed
  - why
  - compatibility impact
  - validation method

## Documentation Style

- Keep section numbering consistent with existing files
- Preserve bilingual style where already used
- Prefer explicit definitions over metaphor when possible
